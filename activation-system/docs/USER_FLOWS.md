## 1. Администратор (Admin)

**Роль:** Полный контроль над системой, управление всеми справочниками, пользователями, финансами и интеграциями.

### 1.1 Дашборд и Аналитика
**Путь:** `/admin/dashboard`

**Действия:**
- Просмотр общей статистики системы (количество пользователей, мерчантов, продаж, ваучеров, клиентов)
- Мониторинг ключевых метрик

---

### 1.2 Управление Вендорами

#### 1.2.1 Просмотр списка вендоров
**Путь:** `/admin/vendors`

**Действия:**
1. Просмотр всех вендоров с информацией:
   - ID, Название, Категория
   - Тип товаров (`ROKKY`, `MANUAL`, `VOUCHER`)
   - Описание
   - Текущий баланс (задолженность)
2. Переход к редактированию, пополнению баланса или истории транзакций

#### 1.2.2 Создание нового вендора
**Путь:** `/admin/add-vendor`

**Флоу:**
1. Нажать **"Добавить вендора"**
2. Заполнить форму:
   - **Название**: Имя вендора (например, "Spotify")
   - **Категория**: (например, "Music", "Gaming")
   - **Тип товаров**:
     - `ROKKY`: Автоматическая активация через API Rokky
     - `MANUAL`: Ручная обработка оператором
     - `VOUCHER`: Ваучеры, активируемые на сторонних ресурсах
   - **Описание**: Краткое описание
   - **Комиссия по умолчанию**: % комиссии для новых продуктов
   - **Шаблон чека**: JSON-схема для генерации чеков (опционально)
3. Нажать **"Сохранить"**

#### 1.2.3 Редактирование вендора
**Путь:** `/admin/vendors/edit/:id`

**Действия:**
- Изменение всех параметров вендора
- Редактирование шаблона чека с предпросмотром
- Сохранение изменений

#### 1.2.4 Пополнение баланса вендора
**Путь:** `/admin/vendor/:id/pay`

**Флоу:**
1. Открыть форму пополнения
2. Ввести сумму платежа
3. Указать комментарий (опционально)
4. Подтвердить операцию
5. Баланс вендора обновляется, создается запись в истории платежей

#### 1.2.5 Просмотр транзакций вендора
**Путь:** `/admin/vendor/:id/transactions`

**Действия:**
- Просмотр всех транзакций (продажи, начисления, списания)
- Фильтрация по статусу и дате

#### 1.2.6 Создание пользователя вендора
**Путь:** `/admin/add-vendor-users`

**Флоу:**
1. Выбрать вендора из списка
2. Ввести данные пользователя:
   - Username
   - Password
   - Примечание (опционально)
3. Создать пользователя с ролью `vendor_user`

---

### 1.3 Управление Продуктами

#### 1.3.1 Просмотр списка продуктов
**Путь:** `/admin/products`

**Действия:**
- Просмотр всех продуктов с информацией:
  - ID, Название, Цена, Статус
  - Количество активных ваучеров
  - Наличие кастомного шаблона чека
- Переход к редактированию или удалению

#### 1.3.2 Создание нового продукта
**Путь:** `/admin/add-product`

**Флоу:**
1. Нажать **"Добавить новый товар"**
2. Заполнить форму:
   - **Название**: Название продукта
   - **Цена**: Цена в сумах
   - **Статус**: `on` (активен) / `off` (неактивен)
   - **Вендор**: Выбрать из списка
   - **Комиссия мерчанта**: % от цены
   - **Комиссия вендора**: % от цены
   - **Rokky SKU**: Для товаров типа ROKKY (опционально)
3. Нажать **"Создать товар"**

#### 1.3.3 Редактирование продукта
**Путь:** `/admin/edit-product/:id`

**Действия:**
- Изменение всех параметров продукта
- Настройка кастомного шаблона чека (если нужен отличный от вендора)
- Предпросмотр шаблона чека
- Сохранение изменений

#### 1.3.4 Удаление продукта
**Путь:** `POST /admin/delete-product/:id`

**Важно:** 
- Удаление **невозможно**, если есть связанные ваучеры или продажи
- Система выведет ошибку: *"Требуется удаление ваучеров и истории продаж перед удалением товара"*
- Это защищает целостность данных и взаиморасчеты

### 1.4 Управление Ваучерами

#### 1.4.1 Просмотр списка ваучеров
**Путь:** `/admin/vouchers`

**Действия:**
- Просмотр всех ваучеров в системе
- Фильтрация по статусу, продукту, вендору

#### 1.4.2 Добавление одного ваучера
**Путь:** `/admin/add-voucher`

**Флоу:**
1. Ввести код ваучера
2. Выбрать продукт
3. Тип ваучера устанавливается автоматически (`Vendor`)
4. Нажать **"Создать ваучер"**

#### 1.4.3 Массовое добавление ваучеров
**Путь:** `/admin/vouchers/add`

**Флоу:**
1. Выбрать продукт
2. Ввести список кодов (по одному на строку)
3. Нажать **"Добавить"**
4. Система создаст все ваучеры, пропуская дубликаты

#### 1.4.4 Генерация ваучеров
**Путь:** `POST /admin/vouchers/generate`

**Флоу:**
1. Выбрать продукт
2. Указать серию (например, "AB")
3. Указать количество (1-100,000)
4. Нажать **"Сгенерировать и добавить"**
5. Система создаст уникальные коды формата `{СЕРИЯ}{6 случайных символов}`

### 1.11 Обработка Ручных Активаций (Manual Activations)

#### 1.11.1 Список запросов
**Путь:** `/admin/manual-activations`

**Действия:**
- Просмотр всех запросов на ручную активацию
- Фильтрация по статусу (`PENDING`, `COMPLETED`, `REJECTED`)
- Фильтрация по магазину

#### 1.11.2 Детали запроса
**Путь:** `/admin/manual-activations/:id`

**Действия:**
- Просмотр полной информации о запросе:
  - **ID заявки** (отображается в Telegram уведомлении как `#ID`)
  - Код ваучера
  - Продукт
  - Клиент (телефон)
  - Дата создания
  - Статус

**Примечание:** При создании новой заявки на ручную активацию, оператор получает уведомление в Telegram с указанием ID заявки (например, `#42`), что упрощает поиск и обработку запроса в админ-панели.

#### 1.11.3 Завершение активации
**Путь:** `POST /admin/manual-activations/:id/complete`

**Флоу:**
1. Получить уведомление в Telegram с **ID заявки** (например, `#42`)
2. Перейти в **Admin Panel** → **Manual Activations**
3. Найти запрос по ID или статусу `PENDING`
4. Получить код активации от вендора (вручную, по телефону/email)
5. Ввести ключ активации в форму
6. Нажать **"Завершить активацию"**
7. Система:
   - Обновляет статус ваучера на `activated`
   - Создает запись активации
   - Отправляет SMS клиенту с уведомлением
   - Обновляет транзакции

#### 1.11.4 Отклонение запроса
**Путь:** `POST /admin/manual-activations/:id/reject`

**Флоу:**
1. Указать причину отклонения
2. Нажать **"Отклонить"**
3. Система:
   - Возвращает ваучер в статус `sold`
   - Удаляет привязку к клиенту
   - Запрос помечается как `REJECTED`

## 2. Мерчант (Merchant)

**Роль:** Продажа ваучеров конечным клиентам в точках продаж.

### 2.1 Дашборд

**Путь:** `/merchant/dashboard`

**Действия:**
- Просмотр статистики продаж
- Информация о текущей задолженности
- Быстрый доступ к основным функциям

---

### 2.2 Продажа Ваучеров

#### 2.2.1 Выбор товаров
**Путь:** `/merchant/sell`

**Флоу:**
1. Просмотр доступных товаров
2. Выбор товара
3. Нажать **"Добавить в корзину"**

#### 2.2.2 Корзина
**Путь:** `/merchant/checkout`

**Действия:**
- Просмотр товаров в корзине
- Изменение количества
- Удаление товаров
- Переход к оформлению

#### 2.2.3 Оформление продажи (Оффлайн)
**Путь:** `POST /merchant/checkout/confirm`

**Флоу:**
1. Выбрать тип продажи: **"Оффлайн"**
2. Нажать **"Оформить"**
3. Система:
   - Резервирует ваучер (статус `sold`)
   - Генерирует PDF-чек с QR-кодом
   - Автоматически открывает диалог печати
4. Распечатать чек и отдать клиенту
5. Клиент сканирует QR-код для активации

#### 2.2.4 Оформление продажи (Онлайн)
**Путь:** `POST /merchant/checkout/confirm`

**Флоу:**
1. Выбрать тип продажи: **"Онлайн"**
2. Ввести номер телефона клиента (`+998XXXXXXXXX`)
3. Нажать **"Оформить"**
4. Система:
   - Создает/находит клиента по номеру
   - Резервирует ваучер (статус `sold`)
   - Привязывает ваучер к аккаунту клиента
   - Отправляет SMS со ссылкой на активацию
5. Клиент получает SMS и может активировать ваучер в своем кошельке

## 3. Вендор (Vendor User)

**Роль:** Представитель поставщика услуг, управление своими продуктами и активациями.

### 3.1 Дашборд

**Путь:** `/vendor/dashboard`

**Действия:**
- Просмотр статистики по своему вендору
- Количество активных ваучеров
- Количество активаций
- Текущий баланс

---

### 3.2 Активация Ваучеров

**Путь:** `/vendor/activate`

**Флоу (только для вендоров типа НЕ-ROKKY):**
1. Ввести код ваучера
2. Нажать **"Активировать"**
3. Система:
   - Проверяет принадлежность ваучера вендору
   - Проверяет статус (`sold` или `pending`)
   - Для ROKKY вендоров: выдает ошибку (активация только через клиентский магазин)
   - Для других: меняет статус на `activated`
   - Создает запись активации
   - Отправляет SMS клиенту (если привязан)

**Важно:** Ваучеры ROKKY-вендоров **нельзя** активировать через эту форму.

---

### 3.3 Список Ваучеров

**Путь:** `/vendor/vouchers`

**Действия:**
- Просмотр всех ваучеров своего вендора

### 3.4 Транзакции

- Список пользователей вендора

## 4. Клиент (Client / End User)

**Роль:** Покупатель, активирующий и управляющий своими ваучерами.

### 4.1 Аутентификация

#### 4.1.1 Вход в систему
**Путь:** `/wallet`

**Флоу:**
1. Ввести номер телефона (`+998XXXXXXXXX`)
2. Нажать **"Войти"**
3. Получить SMS с 6-значным кодом
4. Ввести код на странице `/client-verify`
5. Попадание в личный кабинет

#### 4.1.2 Регистрация
**Путь:** `/client-register`

**Флоу:**
1. Ввести номер телефона
2. Ввести имя

### 4.3 Активация Ваучера

**Процесс зависит от типа вендора:**

#### 4.3.1 Тип ROKKY (Автоматическая)
**Флоу:**
1. Открыть ваучер в кошельке
2. Нажать **"Активировать"**
3. Система отправляет запрос в Rokky API
4. **Результат (мгновенно):**
   - На экране отображается код активации (Key/Pin)
   - Статус меняется на `Activated`
   - Клиент использует полученный ключ в сервисе вендора

#### 4.3.2 Тип MANUAL (Ручная)
**Флоу:**
1. Нажать **"Активировать"**
2. Система создает запрос на ручную активацию
3. **Результат:**
   - Клиент видит сообщение: *"Заявка принята. Ожидайте SMS с ключом активации в течение нескольких минут"*
   - Оператор получает уведомление в Telegram
   - Оператор обрабатывает запрос вручную
   - После обработки клиент получает SMS с кодом
   - Статус в кошельке обновляется на `Activated`

#### 4.3.3 Тип VOUCHER (Сторонняя активация)
**Флоу:**
1. Нажать **"Активировать"**
2. **Результат:**
   - Ваучер помечается как `Activated` в системе
   - Клиент видит полный код ваучера
   - Клиент самостоятельно использует код на сайте вендора (например, spotify.com/redeem)
   - Наша система только фиксирует факт использования

---

### 4.4 Активация через Магазин (Store)

**Путь:** `/:storeSlug` (например, `/namo`)

**Флоу:**
1. Клиент переходит по ссылке из QR-кода или SMS
2. Если не авторизован:
   - Ввести номер телефона
   - Получить и ввести OTP-код
3. Ввести код ваучера
4. Нажать **"Активировать"**
5. Процесс активации зависит от типа вендора (см. выше)
6. Результат отображается на экране магазина

## Дополнительные Примечания

### Типы Вендоров и их Влияние на Флоу

1. **ROKKY (Автоматическая активация)**
   - Активация происходит мгновенно через API
   - Клиент получает ключ сразу на экране
   - Требуется настройка Rokky SKU для продукта
   - Нельзя активировать через форму вендора

2. **MANUAL (Ручная активация)**
   - Создается запрос оператору
   - Оператор получает уведомление в Telegram
   - Оператор вручную вводит ключ в админ-панели
   - Клиент получает SMS с ключом

3. **VOUCHER (Сторонняя активация)**
   - Система только фиксирует факт использования
   - Клиент сам активирует код на сайте вендора
   - Не требуется интеграция с API

### Защита Данных

- **Удаление продуктов**: Невозможно при наличии связанных ваучеров или продаж
- **Удаление вендоров**: Невозможно при наличии продуктов
- **Целостность транзакций**: Все финансовые операции защищены от случайного удаления

### SMS Уведомления

SMS отправляются в следующих случаях:
- Онлайн продажа (ссылка на активацию)
- Завершение ручной активации (ключ активации)
- Активация через магазин (подтверждение)

### Роли и Иерархия

```
Admin
  ├── Полный доступ ко всем функциям
  └── Управление всеми пользователями

Merchant
  ├── Продажа ваучеров
  └── Просмотр своих продаж

Vendor User
  ├── Привязан к конкретному вендору
  ├── Активация ваучеров своего вендора
  └── Просмотр статистики своего вендора

Client
  ├── Активация купленных ваучеров
  └── Управление своим кошельком
```
