# Пользовательские флоу и обязанности (User Flows)

В этом документе описаны основные сценарии работы для каждой роли в системе **POSA Activation System**.

---

## 1. Администратор (Admin)

**Роль:** Полный контроль над системой, управление вендорами, мерчантами и пользователями.

### Основные обязанности:
- Управление справочниками (Вендоры, Мерчанты, Продукты).
- Мониторинг транзакций и балансов.
- Обработка ручных запросов на активацию.
- Настройка интеграций (Telegram боты).

### Флоу: Создание нового вендора
1.  Перейти в **Admin Panel** -> **Vendors**.
2.  Нажать **"Добавить вендора"**.
3.  Заполнить форму:
    -   **Название**: Имя вендора (например, "Spotify").
    -   **Категория**: (например, "Music").
    -   **Тип товаров**:
        -   `ROKKY`: Для автоматической активации через API.
        -   `MANUAL`: Для ручной обработки оператором.
        -   `VOUCHER`: Для ваучеров, активируемых на сторонних ресурсах.
    -   **Комиссия**: % комиссии системы.
4.  Нажать **"Сохранить"**.

### Флоу: Обработка ручной активации (Manual Activation)
1.  Получить уведомление (в Telegram или проверить панель).
2.  Перейти в **Admin Panel** -> **Manual Activations**.
3.  Найти запрос со статусом `PENDING`.
4.  Открыть детали запроса.
5.  **Действие**:
    -   **Подтвердить**: Ввести полученный код/ключ активации и нажать "Завершить". Клиент получит SMS.
    -   **Отклонить**: Указать причину отказа. Средства/статус ваучера будут возвращены.

## 2. Мерчант (Merchant)

**Роль:** Продажа ваучеров конечным клиентам (в точках продаж).

### Основные обязанности:
- Продажа товаров (ваучеров).
- Печать чеков.
- Просмотр истории продаж.

### Флоу: Продажа ваучера (Оффлайн)
1.  Войти в систему под учетной записью мерчанта.
2.  Перейти в раздел **"Продажа"** (POS терминал).
3.  Добавить товары в корзину.
4.  Нажать **"Оформить"**.
5.  Выбрать тип продажи **"Оффлайн"**.
6.  Система сгенерирует чек (PDF) с QR-кодом для активации.
7.  Распечатать чек и отдать клиенту.

### Флоу: Продажа ваучера (Онлайн)
1.  В корзине выбрать тип продажи **"Онлайн"**.
2.  Ввести номер телефона клиента (`+998...`).
3.  Нажать **"Оформить"**.
4.  Клиент получит SMS со ссылкой на активацию ваучера.
5.  Ваучер автоматически привязывается к аккаунту клиента (Wallet).

---

## 3. Вендор (Vendor User)

**Роль:** Представитель поставщика услуг (например, Netflix, Spotify).

### Основные обязанности:
- Управление своими продуктами.
- Настройка внешнего вида чеков.
- Просмотр статистики по своим продажам.

### Флоу: Настройка шаблона чека
1.  Войти в **Vendor Dashboard**.
2.  Перейти в **Settings** (Настройки).
3.  В разделе **"Конструктор шаблона чека"**:
    -   Перетащить блоки (Лого, Текст, Список товаров, QR).
    -   Настроить текст и выравнивание.
4.  Проверить **Предпросмотр** справа.
5.  Нажать **"Сохранить изменения"**.
    *Теперь все чеки для товаров этого вендора будут печататься в новом дизайне.*

---

## 4. Клиент (Client / End User)

**Роль:** Покупатель, активирующий ваучер.

### Основные обязанности:
- Активация купленных ваучеров.
- Хранение истории покупок (Wallet).

### Флоу: Вход в систему
1.  Перейти по ссылке из SMS или QR-кода (`wallet.namo.uz`).
2.  Ввести номер телефона.
3.  Ввести код из SMS (OTP).
4.  Попадание в **Личный кабинет (Dashboard)**.

### Флоу: Активация ваучера
Процесс зависит от типа вендора (`productType`):

#### А. Тип ROKKY (Автоматическая)
1.  Клиент открывает ваучер в кошельке.
2.  Нажимает **"Активировать"**.
3.  Система отправляет запрос в Rokky API.
4.  **Результат**: На экране мгновенно отображается код активации (Key/Pin). Статус меняется на `Activated`.

#### Б. Тип MANUAL (Ручная)
1.  Клиент нажимает **"Активировать"**.
2.  Система создает заявку оператору.
3.  **Результат**: Клиент видит сообщение "Заявка принята".
4.  После обработки оператором, клиент получает SMS с кодом, и статус в кошельке обновляется.

#### В. Тип VOUCHER (Сторонняя)
1.  Клиент нажимает **"Активировать"**.
2.  **Результат**: Ваучер помечается как использованный (`Activated`) в нашей системе.
3.  Клиент использует сам код ваучера (который был на чеке/в смс) на официальном сайте сервиса (например, вводит код на spotify.com/redeem).
    *(Наша система здесь выступает только как реестр продаж).*
