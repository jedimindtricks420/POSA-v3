# Руководство по ручным активациям (Manual Vendor)

## Обзор
Система поддерживает два типа активации:
1. **Автоматическая (Rokky)** - ключи получаются через API автоматически
2. **Ручная (Manual)** - оператор вручную вводит ключи активации

## Настройка

### 1. Создание Telegram бота
1. Напишите @BotFather в Telegram
2. Создайте нового бота командой `/newbot`
3. Скопируйте токен бота (например: `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`)
4. Узнайте свой Telegram ID у @userinfobot

### 2. Настройка бота в админ-панели
1. Перейдите в **Admin Panel → Telegram Боты**
2. Нажмите **"Создать бота"**
3. Заполните форму:
   - **Название**: например "Microsoft Notifications"
   - **Токен бота**: вставьте токен от BotFather
   - **Telegram ID пользователей**: введите ID операторов через запятую (например: `123456789, 987654321`)
   - **Привязанные магазины**: выберите магазины, от которых будут приходить уведомления
   - **Активен**: поставьте галочку
4. Нажмите **"Создать"**
5. Нажмите **"Тест"** чтобы проверить, что бот работает

### 3. Создание вендора с типом Manual
1. Перейдите в **Admin Panel → Вендоры → Добавить вендора**
2. В поле **"Тип продукта"** выберите или введите `Manual`
3. Заполните остальные поля и сохраните

### 4. Создание продукта
1. Создайте продукт для вендора с типом `Manual`
2. Привяжите продукт к магазину (например, `/kaspersky`)

### 5. Настройка SMS уведомления (опционально)
1. Перейдите в **Admin Panel → Магазины**
2. Выберите нужный магазин и нажмите **"Изменить"**
3. В поле **"Текст SMS уведомления"** введите свой текст
4. По умолчанию: `Vash vaucher aktivirovan | Sizning vaucheringiz faollashtirildi https://namo.uz/link`

## Процесс активации

### Шаг 1: Клиент вводит ваучер
1. Клиент заходит на страницу магазина (например: `https://your-domain.com/kaspersky`)
2. Вводит свой номер телефона и подтверждает по OTP
3. Вводит код ваучера
4. Получает сообщение: "Ваша заявка принята. Ожидайте SMS с ключом активации в течение нескольких минут."

### Шаг 2: Оператор получает уведомление
В Telegram боте приходит сообщение:
```
🔔 Новая заявка на активацию

Магазин: Kaspersky
Товар: Kaspersky Total Security 1 год
Код ваучера: XXXX-XXXX-XXXX

Клиент:
📱 Телефон: +998901234567
📅 Дата: 26.11.2025 13:05

Для обработки перейдите в админ-панель
```

### Шаг 3: Оператор обрабатывает заявку
1. Оператор заходит в **Admin Panel → Ручные активации**
2. Видит список всех заявок со статусом "Ожидание"
3. Нажимает **"Обработать"** на нужной заявке
4. Вводит ключ активации (например: `XXXX-XXXX-XXXX-XXXX`)
5. Нажимает **"Отправить"**

### Шаг 4: Клиент получает ключ
1. Клиенту приходит SMS с текстом из настроек магазина
2. Ключ появляется в истории на странице магазина
3. Статус заявки меняется на "Завершено"

## Фильтрация заявок
В разделе **"Ручные активации"** можно фильтровать по:
- **Статусу**: Все / Ожидание / Завершено / Отклонено
- **Магазину**: выбрать конкретный магазин

## Отклонение заявки
Если нужно отклонить заявку:
1. Нажмите **"Отклонить"** на заявке
2. Подтвердите действие
3. Ваучер вернется в статус "sold" и клиент сможет попробовать снова

## Управление ботами
- **Редактировать**: изменить настройки бота, добавить/удалить магазины или операторов
- **Удалить**: удалить бота (все привязки к магазинам будут удалены)
- **Тест**: отправить тестовое сообщение для проверки работы бота

## Технические детали
- База данных обновлена с новыми таблицами: `TelegramBot`, `StoreTelegramBot`, `ManualActivationRequest`
- Один бот может обслуживать несколько магазинов
- Один магазин может иметь несколько ботов
- Уведомления отправляются всем авторизованным пользователям бота
- SMS отправляется через существующий сервис Eskiz

## Совместимость
✅ Не влияет на работу Rokky активаций
✅ Не влияет на работу Telegram ваучеров
✅ Не влияет на работу обычных Vendor ваучеров
✅ Полностью обратно совместимо

## Масштабируемость
- Можно создать неограниченное количество ботов
- Можно создать неограниченное количество вендоров типа Manual
- Можно добавить неограниченное количество операторов в каждый бот
