# Отчет по балансам, долгам и комиссиям

В данном документе описаны страницы пользовательского интерфейса (Frontend), на которых отображается информация о финансовых балансах и долгах участников системы, а также места настройки комиссий для товаров и вендоров.

## 1. Страницы с информацией о балансах и долгах

В системе POSA балансы участников имеют разное значение в зависимости от их роли:
- **Мерчант (Merchant):** Баланс представляет собой долг мерчанта перед платформой за проданные товары.
- **Вендор (Vendor):** Баланс представляет собой сумму, которую платформа должна выплатить вендору за его товары.
- **Касса (Kassa):** Баланс представляет собой средства, аккумулированные через онлайн-платежи, ожидающие распределения.

| :--- | :--- | :--- | :--- |

| **Редактирование мерчанта** | `/admin/merchants/edit/:id` | Админ | Детальная информация о мерчанте, включая "Актуальный долг". |

## 2. Страницы настройки комиссий

Комиссии определяют, какая часть стоимости товара уходит платформе, какая — вендору, и какой процент прибыли получает мерчант.

| Название страницы | Относительный путь (URL) | Поле в интерфейсе | Описание |
| :--- | :--- | :--- | :--- |
| **Добавление товара** | `/admin/products/add` | `Комиссия мерчанта (%)`, `Комиссия вендора (%)` | Установка индивидуальных комиссий для конкретного товара. |
| **Редактирование товара**| `/admin/products/edit/:id` | `Комиссия мерчанта (%)`, `Комиссия вендора (%)` | Изменение комиссий существующего товара. |
| **Добавление вендора** | `/admin/vendors/add` | `Комиссия по умолчанию (%)` | Ставка, которая будет подставляться по умолчанию для новых товаров этого вендора. |
| **Редактирование вендора**| `/admin/vendors/edit/:id` | `Комиссия по умолчанию (%)` | Обновление стандартной ставки вендора. |

## 3. Логика работы комиссий и балансов

Расчеты производятся в момент подтверждения продажи (контроллер `/admin/saleController.js`).

### Формулы расчета:

При продаже товара по цене `Price`:

1.  **Долг мерчанта (Merchant Debt):** Это сумма, которую мерчант должен вернуть системе.
    *   `Merchant Debt = Price * (1 - Merchant Commission % / 100)`
    *   *Пример:* Товар стоит 100 000 сум. Комиссия мерчанта 5%. Мерчант оставляет себе 5 000 сум, а 95 000 сум записываются в его баланс как долг перед системой.

2.  **Выплата вендору (Vendor Payout):** Это сумма, которую платформа должна выплатить поставщику товара.
    *   `Vendor Payout = Price * (1 - Vendor Commission % / 100)`
    *   *Пример:* Товар стоит 100 000 сум. Комиссия вендора (доля платформы) 20%. Вендор получает 80 000 сум, которые прибавляются к его балансу.

3.  **Доход платформы (Platform Margin / Kassa Debt):** Чистая прибыль платформы.
    *   `Platform Margin = Price * (Vendor Commission % / 100)`
    *   *Пример:* При комиссии вендора 20%, доля платформы составляет 20 000 сум.

### Маппинг полей базы данных (Prisma Schema):

| Модель | Поле | Описание |
| :--- | :--- | :--- |
| **Merchant** | `balance` | Текущий долг мерчанта за проданные товары. |
| **Vendor** | `balance` | Сумма, которую платформа задолжала вендору. |
| **Vendor** | `defaultCommissionPercent` | Базовая ставка вендора для новых товаров. |
| **Product** | `merchantCommissionPercent` | Процент, который мерчант оставляет себе как прибыль. |
| **Product** | `vendorCommissionPercent` | Процент, который платформа удерживает себе из выплаты вендору. |
| **Kassa** | `balance` | Средства в кассе от онлайн-продаж. |
| **VoucherTransaction** | `merchantDebt`, `vendorDebt`, `kassaDebt` | Снапшоты рассчитанных сумм для конкретной продажи. |
