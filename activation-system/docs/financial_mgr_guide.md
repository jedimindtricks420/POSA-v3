# Руководство пользователя: Финансовый менеджер (financial_mgr)

Этот документ описывает интерфейс и функциональные возможности системы POSA для пользователей с ролью **Финансовый менеджер**.

## Обзор роли
Роль `financial_mgr` предназначена для сотрудников, ответственных за контроль денежных потоков, взаиморасчеты с мерчантами и вендорами, а также мониторинг выручки платформы. 

---

## 1. Главная панель (Дашборд)
**URL:** `/admin/dashboard`

При входе финансовый менеджер видит адаптированный дашборд, сфокусированный на финансовых показателях:
- **Общая выручка:** Сумма всех продаж в системе.
- **Доход платформы:** Чистая прибыль (маржа), полученная за счет комиссий.
- **Задолженность мерчантов:** Общая сумма, которую мерчанты должны перечислить платформе.
- **Обязательства перед вендорами:** Сумма, которую платформа должна выплатить поставщикам товаров.

---

## 2. Раздел "Финансы"
Доступен через основное меню. Содержит ключевые инструменты для управления взаиморасчетами.

### 2.1. Продажи
**URL:** `/admin/sales`
- Просмотр полного списка совершенных продаж.
- Фильтрация и поиск по ваучерам, продуктам и мерчантам.
- Просмотр статистики: средний чек, количество онлайн/офлайн продаж, общая маржа.

### 2.2. Мерчанты (Дебиторская задолженность)
**URL:** `/admin/merchants`
- Список всех торговых точек (мерчантов) с указанием их текущего баланса (долга).
- **Просмотр транзакций:** История всех продаж мерчанта, формирующих его долг.
- **История платежей:** Список всех фактических поступлений денег от мерчанта.
- **Регистрация оплаты:** Форма для записи внесения денег мерчантом в счет погашения долга.

### 2.3. Взаиморасчеты с Вендорами
**URL:** `/admin/vendors`
- Список поставщиков (вендоров) и остатки на их балансах.
- **Регистрация выплаты:** Оформление факта перечисления денег вендору за реализованный товар.
- **История транзакций:** Детализированный отчет о том, из каких продаж сложился баланс вендора.

---

## 3. Раздел "Кассы"
**URL:** `/admin/kassas`
- Мониторинг балансов виртуальных касс (средства, поступившие через Click, Payme и QR-платежи).
- Контроль сумм, накопленных на разных точках приема платежей.
- *Примечание:* Финансовый менеджер может просматривать список касс, но не может создавать новые или изменять их настройки (это делает администратор).

---

## 4. Финансы интеграций
Финансовый менеджер имеет доступ к финансовым показателям внешних сервисов для сверки данных.

- **Rokky Finance:** `/admin/rokky/finance` — просмотр закупки и реализации товаров через Rokky.
- **Dr.Web Finance:** `/admin/drweb/finance` — мониторинг оборота по продукции Dr.Web.

---

## Ограничения доступа
Для обеспечения безопасности данных финансовому менеджеру **запрещены** следующие действия:
- Создание и редактирование учетных записей пользователей.
- Изменение каталога товаров и генерация ваучеров.
- Настройка технических интеграций (API ключи, боты).
- Ручное изменение статусов активации (Manual Activations).
- Удаление данных о клиентах или продажах.
