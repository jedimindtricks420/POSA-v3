# Client PWA Gap Backlog

## Functional Wallet Experience
- [ ] Реализовать серверные маршруты `/wallet/pass/:id` и `/wallet/google/:id` с генерацией подписанных Apple Wallet `.pkpass` и Google Wallet JWT/objects для ваучеров.
- [ ] Добавить импорт `.pkpass`/Google Wallet пакетов и привязку к клиентам, включая обработку обновлений от эмитентов.
- [ ] Расширить клиентский кабинет: поиск, сортировка, группировка по бренду/эмитенту, архивирование использованных или просроченных ваучеров, восстановление из архива.
- [ ] Ввести версионирование карточек ваучеров с хранением истории изменений и возможностью отката.

## Dynamic Content & Transactions
- [ ] Настроить серверные события/воркеры, которые обновляют статусы ваучеров и push-атрибуты (BRANDING, terms) с доставкой через Web Push/Background Sync.
- [ ] Поддержать Payment Request/Handler API для токенизированных платежей и выдачи цифровых чеков, включая отмену операций и отображение истории транзакций.

## Context & Engagement
- [ ] Реализовать напоминания об истечении, автоматический архив просроченных ваучеров и сценарии повторного уведомления.
- [ ] Добавить геозоны и расписания для контекстных подсказок (например, появление карточки при посещении точки продаж).

## Installability & UX
- [ ] Настроить кастомные подсказки Add to Home Screen, брендированный splash screen и набор иконок для iOS/Android/Desktop.
- [ ] Дополнить `manifest.json` полями `scope`, `categories`, `screenshots`, `shortcuts` с deeplink-params и разделить иконки по плотностям.

## Offline, Sync & Performance
- [ ] Подключить очередь запросов в сервис-воркере (использовать `queueRequest`/Background Sync) и обеспечить оффлайн-операции (claim/log) с последующей синхронизацией.
- [ ] Внедрить Periodic Sync/API polling для тихих обновлений и отображение последнего времени синка.
- [ ] Ввести метрики производительности (Web Vitals, TTI, bundle size budgets) и автоматическую проверку Lighthouse ≥ 95 на CI.

## Accessibility & Localization
- [ ] Провести WCAG 2.2 AA аудит (скринридеры, клавиатурная навигация, контраст) и устранить найденные проблемы.
- [ ] Добавить i18n слой (локализация строк, форматы дат/валют, RTL-поддержка) и настройки персонализации уведомлений.

## Security & Compliance
- [ ] Внедрить WebAuthn/FIDO2 как второй фактор, session binding и повторную верификацию для чувствительных операций.
- [ ] Шифровать данные в IndexedDB/Cache Storage, ввести ротацию ключей и аппаратные хранилища при наличии.
- [ ] Применить HTTPS + HSTS, строгую CSP, COOP/COEP, Subresource Integrity и защиту от CSRF/XSS на уровне Express.
- [ ] Описать и реализовать процессы соответствия GDPR/CCPA/PDPA (consent management, экспорт/удаление данных), а также PCI DSS/PSD2 для платежных сценариев.

## Integrations & Notifications
- [ ] Предоставить API/вебхуки для эмитентов (JWT/JWS-подписи, SLA, sandbox) и документацию.
- [ ] Реализовать серверную рассылку Web Push с действиями, fallback через SMS/e-mail и управление подписками (удаление, TTL, badge count).
- [ ] Добавить поддержку Web NFC/Smart Tap, защиту экрана от скриншотов и адаптацию яркости/вибросигналов.
- [ ] Провести кросс-браузерные и кросс-платформенные тесты (Chrome, Safari, Edge, Firefox, Android, iOS, Desktop) и задокументировать PWA install mode.

## Observability & Delivery
- [ ] Настроить структурированные логи, сбор метрик (sync latency, push delivery), мониторинг фронтенд-ошибок и алерты SLA < 5 мин.
- [ ] Добавить автоматические тесты (unit, integration, e2e, оффлайн/онлайн сценарии) и pipeline с canary/phased rollout, feature flags и быстрым откатом.
- [ ] Описать план резервного копирования/восстановления (geo-redundant storage, RPO, RTO) и регулярно тестировать восстановление.
- [ ] Организовать регулярные security reviews, SAST/DAST, dependency scanning и политику раскрытия уязвимостей.
- [ ] Подготовить центр поддержки (help center, статус-страница, SLA, обратная связь в приложении) и отчётность по инцидентам.
