### Техническое задание: модуль сканирования QR для PWA-кошелька

---

#### 1. Цель и контекст

- Обеспечить корректную работу сканирования ваучеров в PWA-приложении (iOS/Android/desktop).
- Гарантировать, что камера открывается автоматически, QR-код распознаётся без дополнительных действий, ваучер добавляется либо отображается понятная ошибка.
- Учитывать ограничения Safari/iOS (нет `BarcodeDetector`, `getUserMedia` ограничен, ZXing не должен сам управлять потоком).

---

#### 2. Термины и сокращения

| Термин | Определение |
| --- | --- |
| PWA | Клиентское приложение (страница `/client/qr-scanner`). |
| ZXing | Библиотека `@zxing/browser` (UMD) для распознавания QR. |
| Fallback | Запасной путь распознавания (кадровая обработка canvas + jsQR или аналог). |
| Voucher | Ваучер (`voucher.value`). |
| Claim endpoint | `POST /api/client/voucher/claim` — привязка ваучера к клиенту. |

---

#### 3. Архитектура решения

1. UI: `views/pages/client-qr-scanner.ejs`.
2. Клиентский модуль: `public/js/client-scan.js` (основная логика камеры/сканера).
3. ZXing: UMD-скрипт `node_modules/@zxing/browser/umd/zxing-browser.min.js`, подключается в шаблоне и при необходимости динамически.
4. Fallback: jsQR (или аналог) + Canvas; запускается, если ZXing падает.
5. API: `controllers/client/api/voucherController.js:claim` — основной бекенд.
6. SW: `public/sw.js` — отвечает за кеширование/обновление модулей PWA.

---

#### 4. Функциональные требования

##### 4.1 Инициализация камеры

1. `navigator.mediaDevices.getUserMedia({ video: { facingMode: 'environment', width: { ideal: 1280 }, height: { ideal: 720 } }, audio: false })`.
2. Поток сохраняется в `activeStream`, присваивается `<video>`; устанавливаются `autoplay`, `muted`, `playsinline`.
3. При переключении камеры (`Сменить камеру`) старый поток останавливается, вызывается `getUserMedia` с `facingMode: 'user'` и вся цепочка запускается заново.
4. При `offline` поток освобождается, UI показывает баннер.

##### 4.2 Распознавание (основной путь — ZXing)

1. После `video.play()` создаём offscreen-canvas (`canvas` + `2d` context).
2. В `requestAnimationFrame` копируем кадр: `ctx.drawImage(video, 0, 0, targetWidth, targetHeight)`.
3. Используем `ZXingBrowser.HTMLCanvasElementLuminanceSource` + `ZXing.HybridBinarizer` + `ZXing.BinaryBitmap`.
4. `zxingReader.decode(binaryBitmap)` (с установленными `DecodeHintType.POSSIBLE_FORMATS = QR_CODE`).
5. Исключения `NotFound/Checksum/Format` считаем нормальными, остальные логируем и переключаемся на fallback.
6. Успех → `processPayload(result.getText())`, сканер останавливается до закрытия модалки.

##### 4.3 Fallback (jsQR)

1. Используем тот же canvas, каждые ~250 мс считываем `ImageData`.
2. `jsQR(imageData.data, width, height, { inversionAttempts: 'attemptBoth' })`.
3. При успехе → `processPayload(qr.data)`.
4. При отсутствии jsQR либо нераспознанном коде → пользователь вводит код вручную.

##### 4.4 Обработка результата

1. `processPayload` принимает строку (код, URL с `voucher`, JSON-пакет). Используется `utils/qr.js:extractVoucherCode`.
2. Проверяем дубликаты (в течение 2 секунд игнорируем тот же `payload`).
3. `claimVoucher(payload)` → backend:
   - Успех: заполняем модалку, сохраняем историю (`walletOfflineStore.appendScanHistory`).
   - 4xx: показываем текст ошибки (`Ваучер уже привязан...`).
   - Ошибка сети: alert «Не удалось обработать ваучер. Попробуйте позже» + опция ручного ввода.

##### 4.5 UI/UX

- Камера открывается автоматически (после разрешения).
- История последних сканов (до 5 записей), кнопка «Очистить» очищает IndexedDB.
- При нажатии вне модалки — модалка закрывается, сканер перезапускается.
- Баннер `offline`/`online` отображается в шапке экрана.

##### 4.6 Сервис-воркер

- `public/sw.js` содержит актуальную `CACHE_VERSION` и список `PRECACHE_URLS`, включая `client-scan.js`, `zxing-browser.min.js`, `sw-update.js`.
- После деплоя PWA должна автоматически обновляться (skipWaiting/clientsClaim).

---

#### 5. Обработка ошибок

| Сценарий | Поведение |
| --- | --- |
| Пользователь отклонил доступ к камере | Сообщение «Не удалось получить доступ к камере. Проверьте разрешения.»; сканер не запускается. |
| ZXing выбрасывает ошибку при старте | Переключаемся на fallback (canvas + jsQR). |
| ZXing/jsQR не нашли код в течение 30 сек. | Открываем уведомление с предложением ввести код вручную. |
| `/claim` вернул 409/404 | Модалка с текстом backend (`Ваучер уже привязан...`). |
| Неподдерживаемое устройство | Camera view остаётся пустым, сообщение «Сканер недоступен. Введите код вручную.» |

---

#### 6. Тестирование

1. **Функциональное**: iOS Safari/Chrome, Android Chrome/Firefox, desktop Safari/Chrome.
2. **Переключение камер**: минимум 3 смены подряд — поток не должен зависать, фонарик работает (если поддерживается).
3. **Повторные сканы**: 10 последовательных QR → дубляжи не допускаются, история корректна.
4. **Offline**: отключить сеть → сканер блокируется, ручной ввод доступен, кэш не повреждается.
5. **Безопасность**: QR не должен содержать секретов (только код/URL), итоговый запрос `/claim` защищён логикой backend.

#### 7. План внедрения

1. Переписать `public/js/client-scan.js` согласно требованиям (ручное `getUserMedia`, `decodeFromVideoElementContinuously`, fallback jsQR).
2. Подключить/добавить jsQR (или выбранную библиотеку) в PWA; добавить файл в precache.
3. Обновить шаблон `client-qr-scanner.ejs`: подключить UMD ZXing с `data-zxing-umd`, удалить лишние скрипты.
4. Обновить `public/sw.js` (версия кеша + новый список файлов), проверить `sw-update.js`.
5. Дополнить README-PWA.md разделом о разрешениях камеры/обновлении PWA.
6. Пройти тестовую матрицу (устройства, кейсы). Зафиксировать результаты.
7. Деплой + мониторинг логов (проверка ошибок ZXing/jsQR).

---

#### 8. Риски/ограничения

- iOS требует HTTPS и пользовательское действие (PWA должен быть установлен/открыт после клика).
- Старые iOS (до 15) могут не поддерживать `createImageBitmap`; нужно fallback через `canvas.getContext('2d')`.
- jsQR устойчив хуже ZXing — качество QR на чековой бумаге должны обеспечивать сами продавцы.
- Высокая нагрузка (много `ImageData`) может ухудшить FPS — при необходимости переносим распознавание в WebWorker.

---

#### 9. Результат

- Камера открывается на всех поддерживаемых устройствах.
- QR распознаётся автоматически (ZXing) либо через fallback.
- Ваучер добавляется или отображается чёткая причина отказа.
- В случае проблем пользователь всегда может ввести код вручную.
- PWA автоматически подтягивает свежие версии файлов (без ручного сброса кеша).
