# ТЕХНИЧЕСКОЕ ЗАДАНИЕ: Страницы оплаты товаров

## 1. ОБЩЕЕ ОПИСАНИЕ

Система публичных страниц оплаты предназначена для покупки цифровых ваучеров клиентами без авторизации. Доступ к странице осуществляется по уникальному токену (обычно через QR-код).

## 2. СТРУКТУРА ПРОЦЕССА ОПЛАТЫ

Процесс оплаты состоит из 4 страниц:

1. **Страница товара** (`pay-product.ejs`) - точка входа
2. **Страница оформления** (`pay-checkout.ejs`) - сбор данных для оплаты
3. **Страница результата** (`pay-result.ejs`) - успешная оплата
4. **Страница ошибки** (`pay-error.ejs`) - обработка ошибок

---

## 3. СТРАНИЦА ТОВАРА (pay-product.ejs)

### 3.1 Маршрут
- URL: `/pay/:token`
- Метод: `GET`

### 3.2 Входные данные (переменные EJS)

```javascript
{
  product: {
    name: String,        // Название товара
    price: Number        // Цена товара в сумах
  },
  merchant: {
    username: String     // Имя продавца
  },
  hasVouchers: Boolean,  // Наличие ваучеров в наличии
  link: {
    token: String        // Токен ссылки для навигации
  }
}
```

### 3.3 Функциональные требования

1. **Отображение информации о товаре:**
   - Логотип/иконка вендора (по умолчанию эмодзи 🎵)
   - Название товара
   - Цена с форматированием (разделение тысяч)
   - Информация о продавце

2. **Управление доступностью:**
   - Если `hasVouchers === true`:
     - Отобразить активную кнопку "КУПИТЬ"
     - При клике переход на `/pay/:token/checkout`
   - Если `hasVouchers === false`:
     - Отобразить неактивную кнопку
     - Текст: "Товар временно недоступен"

3. **Адаптивность:**
   - Страница должна быть оптимизирована для мобильных устройств
   - Viewport: `width=device-width, initial-scale=1`

## 4. СТРАНИЦА ОФОРМЛЕНИЯ (pay-checkout.ejs)

### 4.1 Маршрут
- URL: `/pay/:token/checkout`
- Метод: `GET`

### 4.2 Входные данные (переменные EJS)

```javascript
{
  product: {
    name: String,
    price: Number
  },
  token: String  // Токен для API-запросов и навигации
}
```

### 4.3 Функциональные требования

#### 4.3.1 Навигация
- Кнопка "← Назад" с переходом на `/pay/:token`

#### 4.3.2 Информационный блок
- Отображение названия товара
- Отображение цены с форматированием

#### 4.3.3 Форма ввода данных

**Поле "Номер телефона":**
- Тип: `tel`
- Префикс: `+998` (нередактируемый)
- Плейсхолдер: "90 123 45 67"
- Валидация:
  - Только цифры
  - Ровно 9 символов
  - Pattern: `[0-9]{9}`
  - Required: true
- Подсказка: "На этот номер будет отправлен ваучер"

**Выбор способа оплаты:**
- Тип: radio buttons
- Визуализация: карточки с иконками
- Варианты:
  1. Click (выбран по умолчанию)
  2. Payme
- Интерактивность:
  - При выборе - подсветка границы синим цветом
  - Изменение фона на светло-синий (bg-blue-50)

**Итоговая сумма:**
- Отображение строки "Итого к оплате:" с суммой

**Кнопка оплаты:**
- Текст: "ОПЛАТИТЬ {price} сум"
- Тип: submit
- Стиль: зелёная, полная ширина

**Согласие:**
- Текст: "Нажимая 'Оплатить', вы соглашаетесь с условиями оферты"
- Размер шрифта: мелкий
- Цвет: серый

#### 4.3.4 Состояние загрузки

При отправке формы:
1. Скрыть форму
2. Показать экран загрузки с:
   - Иконкой ⏳
   - Текстом "Обработка платежа..."
   - Предупреждением "Пожалуйста, не закрывайте страницу"

#### 4.3.5 JavaScript функциональность

**Валидация перед отправкой:**
1. Проверить, что введено ровно 9 цифр
2. Если нет - показать alert: "Введите 9 цифр номера телефона"

**Обработка отправки формы:**
1. Метод: `POST`
2. URL: `/pay/:token/checkout`
3. Content-Type: `application/json`
4. Тело запроса:
```javascript
{
  phoneNumber: String,    // "+998" + 9 цифр
  paymentMethod: String   // "click" или "payme"
}
```

5. Обработка ответа:
   - Успех (`data.success === true` и есть `data.redirectUrl`):
     - Выполнить редирект на `data.redirectUrl`
   - Ошибка:
     - Показать alert с текстом ошибки
     - Вернуть форму на экран
     - Скрыть экран загрузки

6. Обработка сетевых ошибок (catch):
   - Показать alert: "Ошибка соединения"
   - Вернуть форму на экран

**Визуальная обратная связь:**
- Подсветка выбранного способа оплаты
- Автоматическая подсветка первого варианта при загрузке

## 5. СТРАНИЦА РЕЗУЛЬТАТА (pay-result.ejs)

### 5.1 Маршрут
- URL: `/pay/:token/result/:attemptId`
- Метод: `GET`

### 5.2 Входные данные (переменные EJS)

```javascript
{
  merchant: {
    username: String,
    legalInfo: String | null    // Опционально: юридическая информация
  },
  product: {
    name: String,
    price: Number
  },
  attempt: {
    id: Number,
    phoneNumber: String,
    paidAt: Date               // Дата и время оплаты
  },
  sale: {
    id: Number                 // Номер продажи
  },
  voucherCode: String,         // Код активации ваучера
  token: String                // Токен для навигации
}
```

### 5.3 Функциональные требования

#### 5.3.1 Заголовок успеха
- Иконка: ✅
- Текст: "Оплата успешна!"
- Цвет: зелёный

#### 5.3.2 Электронный чек

**Заголовок секции:**
- Текст: "ЭЛЕКТРОННЫЙ ЧЕК"
- Оформление: пунктирная рамка

**Информация о продавце:**
- Имя продавца
- Юридическая информация (если есть)

**Информация о товаре:**
- Название товара
- Количество: "1 шт"
- Цена с форматированием

**Итоговая сумма:**
- Текст: "ИТОГО"
- Сумма с форматированием
- Выделение жирным шрифтом

**Детали транзакции:**
- Дата (формат: DD.MM.YYYY, локаль: ru-RU)
- Время (формат: HH:MM, локаль: ru-RU)
- Номер продажи (#ID)
- Номер телефона покупателя

**Код ваучера:**
- Фон: светло-зелёный (green-50)
- Граница: зелёная (green-200)
- Заголовок: "КОД ВАШЕГО ВАУЧЕРА"
- Код:
  - Моноширинный шрифт
  - Крупный размер (text-2xl)
  - Жирный стиль
  - Цвет: тёмно-зелёный (green-700)
  - Увеличенное межбуквенное расстояние (tracking-wider)

#### 5.3.3 Действия

**Кнопка "Сохранить чек":**
- Текст: "📥 СОХРАНИТЬ ЧЕК (PDF)"
- URL: `/pay/:token/receipt/:attemptId`
- Стиль: синяя, полная ширина
- Действие: скачивание PDF-файла

**Кнопка "Открыть кошелёк":**
- Текст: "📱 ОТКРЫТЬ КОШЕЛЁК"
- URL: `/client/dashboard`
- Стиль: серая, полная ширина

#### 5.3.4 Инструкция по использованию

**Заголовок:** "Как использовать ваучер?"

**Шаги (нумерованный список):**
1. Откройте кошелёк wallet.namo.uz
2. Войдите по номеру телефона
3. Найдите ваучер в списке
4. Нажмите "Активировать"

## 6. СТРАНИЦА ОШИБКИ (pay-error.ejs)

### 6.1 Маршрут
- URL: `/pay/:token/error`
- Метод: `GET`
- Query параметры: `?error=<error_type>`

### 6.2 Входные данные (переменные EJS)

```javascript
{
  error: String,     // Код ошибки: 'expired' | 'failed' | 'no_vouchers' | другое
  message: String,   // Опционально: произвольное сообщение об ошибке
  token: String      // Опционально: токен для возврата к товару
}
```

### 6.3 Типы ошибок и их отображение

#### 6.3.1 Ошибка "expired" (Время истекло)
- Иконка: ⏰
- Заголовок: "Время оплаты истекло"
- Цвет заголовка: оранжевый (orange-600)
- Текст:
  ```
  У вас было 2 минуты для завершения оплаты.
  Пожалуйста, попробуйте снова.
  ```

#### 6.3.2 Ошибка "failed" (Платёж отклонён)
- Иконка: ❌
- Заголовок: "Оплата не прошла"
- Цвет заголовка: красный (red-600)
- Текст:
  ```
  Платёж был отклонён. Проверьте баланс карты
  или попробуйте другой способ оплаты.
  ```

#### 6.3.3 Ошибка "no_vouchers" (Товар закончился)
- Иконка: 😔
- Заголовок: "Товар закончился"
- Цвет заголовка: серый (gray-600)
- Текст:
  ```
  К сожалению, пока вы оплачивали,
  последний ваучер был куплен.

  Ваши средства не были списаны.
  ```

#### 6.3.4 Другие ошибки
- Иконка: ❌
- Заголовок: значение переменной `message`
- Цвет заголовка: красный (red-600)

### 6.4 Кнопка возврата

- Отображается, если переменная `token` определена
- Текст: "Попробовать снова"
- URL: `/pay/:token`
- Стиль: синяя кнопка

## 7. ОБЩИЕ ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### 7.1 Технологии
- Шаблонизатор: EJS
- CSS Framework: Tailwind CSS 2.x (через CDN)
- JavaScript: Vanilla JS (ES6+)

### 7.2 Адаптивность
- Все страницы должны корректно отображаться на мобильных устройствах
- Viewport meta tag обязателен
- Максимальная ширина контейнера: `max-w-md` (28rem / 448px)
- Центрирование: `mx-auto`

### 7.3 Форматирование данных

**Числа:**
- Использовать `toLocaleString()` для разделения тысяч

**Даты:**
- Локаль: `ru-RU`
- Формат даты: полный день/месяц/год
- Формат времени: часы:минуты (2-значные)

**Номера телефонов:**
- Префикс: `+998` (нередактируемый)
- Основная часть: 9 цифр без пробелов в базе данных
- Отображение: с пробелами для читаемости (опционально)

### 7.4 Цветовая схема

**Основные цвета:**
- Фон страницы: gray-100
- Карточки: white
- Успех: green-600, green-700
- Кнопка действия: blue-600
- Ошибка: red-600
- Предупреждение: orange-600
- Нейтральный: gray-600

### 7.5 Требования к взаимодействию с API

**Endpoint для оформления:**
- URL: `POST /pay/:token/checkout`
- Content-Type: `application/json`
- Request body:
```json
{
  "phoneNumber": "+998XXXXXXXXX",
  "paymentMethod": "click" | "payme"
}
```

- Success response:
```json
{
  "success": true,
  "redirectUrl": "/pay/:token/result/:attemptId"
}
```

- Error response:
```json
{
  "success": false,
  "error": "Описание ошибки"
}
```

   - Обязательность полей

## 10. СПЕЦИАЛЬНЫЕ СЛУЧАИ

### 10.1 Отсутствие товара
- Проверяется через `hasVouchers`
- Кнопка "КУПИТЬ" становится неактивной
- Текст: "Товар временно недоступен"

### 10.2 Истечение времени оплаты
- Таймаут: 2 минуты с момента создания попытки оплаты
- Редирект на `/pay/:token/error?error=expired`

### 10.3 Отсутствие опциональных данных
- `merchant.legalInfo` - скрывается блок, если значение null
- `token` на странице ошибки - скрывается кнопка "Попробовать снова"
