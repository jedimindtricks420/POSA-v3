## 3. Проверка данных в БД

### 3.1 После генерации ссылки

```sql
SELECT * FROM "MerchantProductLink" ORDER BY id DESC LIMIT 1;
```

| Поле | Проверка |
|------|----------|
| `merchantId` | Корректный |
| `productId` | Корректный |
| `token` | UUID формат |
| `isActive` | true |

### 3.2 После успешной оплаты

```sql
-- Проверить QrPaymentAttempt
SELECT * FROM "QrPaymentAttempt" ORDER BY id DESC LIMIT 1;

-- Проверить Sale
SELECT * FROM "Sale" ORDER BY id DESC LIMIT 1;

-- Проверить Voucher
SELECT id, value, status FROM "Voucher" WHERE value = '<voucher_value>';

-- Проверить OnlineVoucher
SELECT * FROM "OnlineVoucher" ORDER BY id DESC LIMIT 1;

-- Проверить VoucherTransaction
SELECT * FROM "VoucherTransaction" ORDER BY id DESC LIMIT 1;

-- Проверить Client
SELECT * FROM "Client" WHERE "phoneNumber" = '+998XXXXXXXXX';
```

| Таблица | Проверка |
|---------|----------|
| `QrPaymentAttempt` | status = PAID, saleId заполнен |
| `Sale` | saleType = ONLINE, customerPhone заполнен |
| `Voucher` | status = sold |
| `OnlineVoucher` | clientId и voucherId связаны |
| `VoucherTransaction` | merchantDebt, vendorDebt корректны |
| `Client` | Создан или найден существующий |

### 3.3 Проверка балансов

```sql
-- Баланс мерчанта (должен увеличиться)
SELECT id, username, balance FROM "Merchant" WHERE id = <merchant_id>;

-- Баланс вендора (должен увеличиться)
SELECT id, name, balance FROM "Vendor" WHERE id = <vendor_id>;
```

## 4. Edge cases

| # | Сценарий | Ожидаемый результат | ✓ |
|---|----------|---------------------|---|
| 1 | Последний ваучер | Продаётся, следующий покупатель видит "недоступен" |  |
| 2 | Два покупателя одновременно, 1 ваучер | Один успех, один ошибка |  |
| 3 | Повторное открытие result | Чек показывается снова |  |
| 4 | Открытие result чужой попытки | Доступ (TODO: ограничить?) |  |
