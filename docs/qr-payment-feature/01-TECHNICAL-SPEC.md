# Техническое задание: QR-код оплата ваучеров

## 1. Цель

Добавить возможность продажи ваучеров через QR-коды на физических карточках.

## 2. Параметры

| Параметр | Значение |
|---|---|
| Доступность товаров | Все товары всем мерчантам |
| Лимит продаж | Без лимита |
| Типы вендоров | Все (ROKKY, MANUAL, VOUCHER) |
| SMS-уведомления | Нет |
| Интеграция платёжек | Фаза 2 |

---

> [!NOTE]
> **Фаза 1** фокусируется на создании архитектуры: уникальные ссылки для мерчантов, отслеживание попыток оплаты (`QrPaymentAttempt`) и полная цепочка продажи (Sale, Voucher, Client) с эмуляцией платежа.
> **Фаза 2** расширит эту логику за счет подключения реальных API Click и Payme, используя созданную базу.

## 3. Админ-интерфейс

### Страница `/admin/qr-links`

```
┌─────────────────────────────────────────────────────────┐
│  Генерация QR-кодов для оплаты                          │
├─────────────────────────────────────────────────────────┤
│  Мерчант:    [▼ Выберите мерчанта________________]     │
│  Товар:      [▼ Выберите товар___________________]     │
├─────────────────────────────────────────────────────────┤
│           ┌─────────────┐                               │
│           │   QR-КОД    │     Spotify 1 месяц           │
│           └─────────────┘     Цена: 50,000 сум          │
│                               Мерчант: TechStore        │
│  Ссылка: https://wallet.namo.uz/pay/abc123xyz           │
│          [📋 Копировать]  [⬇️ Скачать QR]              │
└─────────────────────────────────────────────────────────┘
```

### Логика

1. Выбор мерчанта → загружается список активных товаров
2. Выбор товара → генерируется/возвращается уникальная ссылка
3. Ссылка уникальна для пары **Мерчант + Товар**
4. QR-код можно скачать как PNG

## 5. Модель данных

### Новые таблицы

```prisma
model MerchantProductLink {
  id          Int       @id @default(autoincrement())
  merchantId  Int
  productId   Int
  token       String    @unique
  isActive    Boolean   @default(true)
  createdAt   DateTime  @default(now())
  
  merchant    Merchant  @relation(fields: [merchantId], references: [id])
  product     Product   @relation(fields: [productId], references: [id])
  paymentAttempts QrPaymentAttempt[]
  
  @@unique([merchantId, productId])
}

model QrPaymentAttempt {
  id                Int       @id @default(autoincrement())
  linkId            Int
  phoneNumber       String
  amount            Float
  paymentMethod     String?
  status            QrPaymentStatus @default(PENDING)
  externalPaymentId String?
  createdAt         DateTime  @default(now())
  expiresAt         DateTime
  paidAt            DateTime?
  saleId            Int?      @unique
  voucherValue      String?
  receiptPath       String?
  
  link              MerchantProductLink @relation(fields: [linkId], references: [id])
  sale              Sale?     @relation(fields: [saleId], references: [id])
}

enum QrPaymentStatus {
  PENDING
  PROCESSING
  PAID
  EXPIRED
  FAILED
}
```

### Изменения существующих

```prisma
model Merchant {
  productLinks MerchantProductLink[]  // добавить
}

model Product {
  merchantLinks MerchantProductLink[]  // добавить
}

model Sale {
  qrPaymentAttempt QrPaymentAttempt?  // добавить
}
```

## 6. Маршруты

### Админ

| Метод | Путь | Назначение |
|---|---|---|
| GET | `/admin/qr-links` | Страница генерации |
| POST | `/admin/qr-links/generate` | AJAX: создать ссылку |
| GET | `/admin/qr-links/:id/download` | Скачать QR PNG |

### Клиент

| Метод | Путь | Назначение |
|---|---|---|
| GET | `/pay/:token` | Страница товара |
| GET | `/pay/:token/checkout` | Форма оплаты |
| POST | `/pay/:token/checkout` | Обработка |
| GET | `/pay/:token/result/:attemptId` | Чек |
| GET | `/pay/:token/receipt/:attemptId` | Скачать PDF |
